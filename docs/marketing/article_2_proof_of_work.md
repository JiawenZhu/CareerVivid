# 工作证明：为什么你的 GitHub 是你的落地页

在产品营销中，**落地页 (Landing Page)** 的唯一目标就是：**转化 (Conversion)**。对于开发者来说，你的 GitHub 仓库和个人作品集就是你的落地页，而你的“转化”就是让 HR 或技术经理点击“发送面试邀请”。

## 1. 拒绝“黑盒同步”

很多开发者在简历上写“精通 Java”，但面试官看不见。你需要提供 **Proof of Work (工作证明)**。

- **不要只隐藏代码**：正如 Austin Kleon 在《秀出你的工作》(Show Your Work!) 中建议的，分享你的过程，而不只是结果。
    - **参考内容**：[Show Your Work! by Austin Kleon](https://austinkleon.com/show-your-work/)
- **文档即产品说明书**：一个没有 README 的仓库就像一个没有说明书的产品。好的 README 应该包含：
    - **Why?** (解决什么问题)
    - **How?** (如何运行)
    - **Screenshots/Gifs** (视觉化你的产出)

## 2. 优化你的“转化路径”

![GitHub 落地页概念](./images/proof_of_work_landing_page.png)

你的 GitHub Profile 应该像一个精心设计的仪表盘：
1. **Pinned Repositories**：这是你的“特色功能区”。挑选 3-4 个最能代表你广度和深度的项目。
2. **Contribution Graph**：这显示了你的“系统可用性 (Uptime)”。持续的绿色代表了你的稳定性和热情。

## 3. 把作品集看作“产品 Demo”

不要只给一个链接。使用像 [CareerVivid](https://careervivid.app) 这样的工具，将你的技术项目包装成具有商业逻辑的故事。

> “代码是廉价的，展示你的思考过程才有价值。”

### 核心行动点：
1. **清理你的 GitHub**：隐藏那些过时的、只有几行代码的练习项目。
2. **编写高质量的 README**：确保别人能在 30 秒内看懂你的项目价值。
